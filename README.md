# JS Photo Blog 📸

## 📌 Descrizione
Un'applicazione web dinamica che simula un blog fotografico, recuperando i contenuti da un'API esterna. Il progetto copre l'intero ciclo di sviluppo: dal testing dell'endpoint alla realizzazione di un'interfaccia responsive alimentata dai dati ricevuti.

## 🎯 Obiettivi

### 1. API Testing & Analysis (Milestone 1-2)
Fase di studio e implementazione della comunicazione con il server.
- **Testing:** Utilizzo di Postman per analizzare la struttura della risposta JSON.
- **Asincronia:** Implementazione di chiamate AJAX tramite `fetch`.
- **Error Handling:** Gestione robusta dei casi critici attraverso i blocchi `.then()`, `.catch()` e `.finally()` per garantire stabilità all'applicazione.

### 2. UI Development (Milestone 3)
Costruzione dell'interfaccia grafica basata sugli asset forniti.
- **Static Layout:** Riproduzione fedele del design tramite HTML e CSS.
- **Component Design:** Creazione di card fotografiche dettagliate e stilizzate.

### 3. Dinamismo & Responsiveness (Milestone 4)
Integrazione finale tra logica e grafica.
- **Data Binding:** Sostituzione dei contenuti statici con i dati reali provenienti dall'API.
- **Responsive Design:** Utilizzo di Media Queries e layout flessibili per adattare la visualizzazione su mobile e tablet, garantendo la leggibilità dei titoli e il corretto incolonnamento delle card.

## 🛠️ Tecnologie Utilizzate
- **HTML5:** Struttura semantica della pagina.
- **CSS3:** Styling avanzato e gestione della responsività.
- **JavaScript (ES6+):** Gestione delle chiamate asincrone e manipolazione dinamica del DOM.
- **Postman:** Strumento per il debug e il test delle API.

---

## 🧪 Flusso di Esecuzione
1. **Analisi:** Verifica dei dati disponibili sull'endpoint fornito.
2. **Sviluppo UI:** Creazione dello scheletro grafico statico.
3. **Integrazione:** Chiamata API al caricamento della pagina e generazione dinamica delle card nel contenitore principale.
4. **Ottimizzazione:** Rifinitura del layout per i diversi breakpoint (dispositivi).
